Course Management System

Tech Stack

1. Frontend

HTML: Used to structure content on the web, including text, images, links, etc.
CSS: Used to define the presentation, layout, and style of HTML elements.
JavaScript: Used to add dynamic content, manipulate HTML elements, handle events, and interact with APIs.
Embedded JavaScript: Used to generate dynamic HTML pages by embedding JavaScript code within HTML templates.

2. Backend

Node.js: For building scalable, server-side applications.
Express.js: Provides a set of tools and middleware for routing, handling requests, and managing server configurations.

3. Database

MongoDB: Used for storing data of faculty, students, admins, courses, scholarships, etc.

4. Middleware

Multer: Used for handling multipart/form-data in Node.js, helpful in uploading files or images.
Express Session: For managing session data on the server side (while logging in).
Bcrypt: For hashing passwords securely (to protect users' passwords).

Features

General Features

Login and Logout features for all users.
Logged-in user ID shown on each page.

Features for Admin

Can add students and faculty.
Can add Core and Elective courses for students.
Can approve all applied scholarships (Will send remarks and status).

Features for Student

Can edit and view details.
Can register for courses and view registration history.
Can add and drop courses for the current semester.
Can submit feedback for each course.
Can view grades.
Can apply for scholarships and view status.

Features for Faculty

Can update grades.
Can approve registered courses (with options of Accepted and Rejected).
Can approve course add/drop (with options of Accepted and Rejected).
Can submit feedback for each course.
Can view submitted feedback.
